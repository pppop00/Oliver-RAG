# 技能 (Skills)

> 本目录用于记录“技能内容”。  
> 若你要修改整个 RAG（如新增 resume/resource/link、变更结构化字段或检索策略），请先阅读根目录 [`SKILLS.md`](../../SKILLS.md)。

这个文件夹记录所有技能的掌握情况和学习历程。

## 内容类型
- ✅ **编程语言**：Python, JavaScript, Java 等
- ✅ **框架和库**：React, TensorFlow, Django 等
- ✅ **工具和平台**：Git, Docker, AWS, Linux 等
- ✅ **方法论**：Agile, TDD, System Design 等
- ✅ **领域知识**：Machine Learning, Data Science, Web Dev 等
- ✅ **软技能**：Leadership, Communication, Project Management 等

## 文件命名规范
```
{skill-name-in-kebab-case}.md
```

示例：
- `python.md`
- `machine-learning.md`
- `react-development.md`
- `technical-leadership.md`

## 使用模板
复制 `../_templates/skill-template.md` 开始创建新的技能文档。

## 熟练度分级
- **beginner**（初学）：基础了解，需要查文档完成任务
- **intermediate**（中级）：能独立完成常规任务
- **advanced**（高级）：能解决复杂问题，理解底层原理
- **expert**（专家）：深度掌握，能教授他人，贡献开源

## 关键要素
每个技能文档应包含：
1. 当前熟练度等级
2. 学习历程和时间线
3. 理论知识掌握情况
4. 实践应用的项目列表
5. 相关认证或证书
6. 技能图谱（前置/相关/衍生技能）
7. 推荐的学习资源

## 技能关联
建立技能之间的联系：
- **前置技能**：学习本技能需要先掌握的
- **相关技能**：互补或经常一起使用的
- **衍生技能**：掌握本技能后容易延伸的

## RAG 优化提示
- 详细记录每个阶段的学习资源和里程碑
- 列出所有应用该技能的项目和工作
- 标注最近使用时间，保持时效性
- 建立技能间的交叉引用
- 包含具体的代码示例或作品链接
