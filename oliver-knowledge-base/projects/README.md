# 项目 (Projects)

这个文件夹存放所有值得记录的项目详情。

## 内容类型
- ✅ **个人项目**：独立完成的side projects
- ✅ **团队项目**：协作完成的项目
- ✅ **工作项目**：职业工作中的重要项目
- ✅ **学术项目**：课程项目、研究项目、毕业设计
- ✅ **开源贡献**：对开源项目的重要贡献
- ✅ **竞赛项目**：Hackathon、编程竞赛项目

## 文件命名规范
```
{project-name-in-kebab-case}.md
```

示例：
- `resume-optimizer-ai.md`
- `real-time-chat-application.md`
- `ml-stock-prediction.md`

## 使用模板
复制 `../_templates/project-template.md` 开始创建新的项目文档。

## 关键要素
每个项目文档应包含：
1. 项目背景和动机
2. 技术架构和技术栈
3. 核心功能和实现细节
4. 量化的成果和影响
5. 遇到的挑战和解决方案
6. GitHub/Demo 链接
7. 学到的技能和经验

## 项目状态
- `completed`：已完成的项目
- `ongoing`：正在进行的项目
- `archived`：已归档/不再维护

## RAG 优化提示
- 详细描述技术实现，包含关键代码片段
- 说明技术选型的理由
- 量化项目影响（用户数、性能指标等）
- 标注所有使用的技术和工具
- 链接到 GitHub 仓库和在线 Demo
